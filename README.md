# 🤖 Маркетинговий Research-Агент — core веб-версія (без новин)

Next.js на Vercel + Neon Postgres. Вхід — **лише номер телефону**, без
PIN/паролю взагалі. Кожна розмова — окрема, з власною історією
(`conversation_id`) — агент бачить тільки повідомлення поточного чату,
ніколи чужі чи старі непов'язані.

Це "очищена" версія: без новинного флоу (`search_telegram_news`,
дедуп новин, перемикач тем) — тільки базовий ресерч-агент. Відповідь
агента — звичайний markdown-текст у чаті, як у стандартному чат-інтерфейсі.

## Архітектура
- `lib/db.js` — Neon Postgres (users / conversations / messages)
- `lib/auth.js` — сесія: httpOnly JWT-кукі
- `lib/tools.js` — інструменти агента (Exa/Apify/Firecrawl/Parallel/SEMrush/безкоштовний analyze_competitor_ads)
- `lib/agent.js` — агентне ядро (function calling), приймає історію КОНКРЕТНОЇ розмови, повертає markdown-текст
- `pages/api/*` — API-роути (auth, conversations, messages, admin)
- `pages/login.js`, `pages/index.js`, `pages/admin.js` — фронтенд

## Крок 1 — Neon (база даних)
1. Зареєструйся на neon.tech, створи проєкт
2. Скопіюй Connection String (postgres://...) → це `DATABASE_URL`
3. Відкрий SQL Editor у Neon і виконай вміст `schema.sql` — створить таблиці users/conversations/messages

## Крок 2 — Вхід (тільки номер телефону)
Юзер вводить номер — і одразу заходить. Номер визначає, під яким акаунтом
він заходить і чиї розмови зберігаються. Жодної перевірки, жодного пароля.

⚠️ **Це НЕ автентифікація особи.** Будь-хто, хто знає адресу сайту, може
ввести БУДЬ-ЯКИЙ номер (включно з чужим) і зайти під ним — побачити чужі
розмови, писати від імені чужого акаунту. Немає жодного механізму, що
підтверджує, що номер телефону справді належить людині, яка його ввела.

Прийнятно тільки для:
- Внутрішнього тесту з довіреною командою
- Особистого MVP, де ти єдиний реальний користувач

**НЕ підходить**, якщо:
- Буде публічний доступ
- В розмовах зберігатимуться чутливі/конфіденційні дані клієнтів
- Кілька незалежних клієнтів користуватимуться одним деплоєм

Якщо потрібен реальний захист — попередня версія проєкту мала або
номер+спільний PIN (проста "перепустка"), або повну SMS-верифікацію через
Twilio Verify. Обидва варіанти можна повернути пізніше без переписування
решти архітектури — доступ ізольований в `pages/api/auth/login.js`.

## Крок 3 — Anthropic + інструменти агента
- `ANTHROPIC_API_KEY` — з console.anthropic.com
- `EXA_API_KEY` — обов'язковий базовий пошук (exa.ai)
- `APIFY_API_KEY`, `FIRECRAWL_API_KEY`, `PARALLEL_API_KEY`, `SEMRUSH_API_KEY` — опційні, вмикають відповідні тули (детальніше — коментарі в `lib/tools.js`)

## Крок 4 — Хто адмін
`ADMIN_PHONES=+380XXXXXXXXX` — свій номер (у форматі, яким логінишся). Можна кілька через кому.
Адмін бачить `/admin` — список усіх юзерів і їхніх розмов (read-only перегляд).

## Крок 5 — Деплой на Vercel
1. Заливаєш цей репозиторій на GitHub
2. Vercel → New Project → імпортуєш репо
3. У Settings → Environment Variables додаєш усі змінні з `.env.example`
4. Deploy

## Локальний запуск
```
npm install
cp .env.example .env.local   # заповнити значеннями
npm run dev
```

## Відомі обмеження MVP
- Вхід без жодного захисту — див. Крок 2, це найважливіше обмеження цієї версії
- Немає live-трейсу "думаю/шукаю" в UI під час відповіді — показується тільки "🧠 Думаю над задачею…"
- `maxDuration: 60` у `vercel.json` — на Vercel Hobby ліміт триваліших функцій може бути нижчим; якщо агент робить багато tool-calls (5-8 кроків × кілька секунд кожен), може не встигнути. Якщо ловиш таймаути — або переходь на Vercel Pro, або зменшуй `MAX_STEPS` у `lib/agent.js`
- Немає rate-limiting на вхід взагалі — при будь-якому публічному доступі це критично, не тільки бажано
